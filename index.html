CREAR TU AVATAR

INSTRUCCIONES

Sigue estos pasos para agregar tu pagina a la galeria.

1

CLONAR EL REPOSITORIO

Abre tu terminal y clona el repositorio del curso:

git clone https://github.com/USUARIO/Teamwork.git cd Teamwork
[TIP] Cambia USUARIO por el nombre del repositorio que te de tu instructor.
2

CREAR UNA RAMA NUEVA

Crea una rama con tu nombre para trabajar sin afectar main:

git checkout -b nombre-apellido
[TIP] Ejemplo: git checkout -b juan-perez
3

CREAR TU PAGINA HTML

Copia la plantilla y renombrala con tu nombre:

cp _plantilla.html nombre-apellido.html

Abre tu archivo y personalizalo: cambia el titulo, escribe sobre ti, agrega tus redes. Se creativo.

[TIP] Revisa FernandoRamirez.html para ver como se ve una pagina terminada.
4

CREAR TU AVATAR 8-BIT

Abre avatar-builder.html en tu navegador. Puedes elegir un preset (spiderman, wizard, ninja, zombie...) o dibujar tu propio personaje pixel por pixel. Personaliza los colores primario y secundario.

[TIP] Los avatares son solo cabezas (8 filas superiores). Si usas un preset, solo copia su nombre. Si dibujas uno, el builder genera el código automáticamente.
5

REGISTRAR TU PAGINA EN LA GALERIA

Abre el archivo team.js y agrega UNA linea con TUS datos dentro del array, debajo del comentario "AGREGA TU LINEA AQUI ABAJO":

{ name: "Tu Nombre", page: "nombre-apellido.html", desc: "Tu frase", avatar: "spiderman", colors: "#e74c3c,#1a1a2e" },

¿Cómo sé que estoy editando MI avatar?
- Busca tu nombre en el array. Si NO estás, agrega una NUEVA línea.
- Si YA estás, edita SOLO tu línea (la que tiene tu nombre).
- NO modifiques las líneas de otros estudiantes.
- Cada línea es independiente = cada estudiante tiene su propio avatar.

[!] Varios alumnos editan este mismo archivo. Si hay un merge conflict, es normal. Solo necesitas conservar tu linea y las de tus companeros. Ver paso 8.
6

COMMIT Y PUSH

Guarda tus cambios y subelos al repositorio:

git add . git commit -m "Agregar pagina de nombre-apellido" git push origin nombre-apellido
7

CREAR UN PULL REQUEST

Ve al repositorio en GitHub. Veras un banner para crear un Pull Request de tu rama. Haz clic en "Compare & pull request", escribe un titulo descriptivo y crealo.

8

RESOLVER MERGE CONFLICTS

Si GitHub indica conflictos al hacer merge, actualiza tu rama:

git checkout main git pull origin main git checkout nombre-apellido git merge main

Abre los archivos con conflictos (probablemente team.js), resuelvelos conservando todas las lineas de tus companeros y la tuya, y sube los cambios:

git add . git commit -m "Resolver merge conflicts" git push origin nombre-apellido
9

MISION COMPLETA

Una vez que tu Pull Request sea aprobado y mergeado, tu pagina aparecera en la galeria del equipo automaticamente. Acabas de colaborar en un proyecto real.
