INSTRUCCIONES

Sigue estos 10 pasos para unirte al proyecto. Todos trabajan sobre el mismo repositorio — cada quien en su propio branch.

01

CLONA EL REPOSITORIO

El instructor te dará acceso al repositorio en GitHub. Una vez que tengas acceso, clónalo a tu computadora:

git clone https://github.com/USUARIO/Teamwork.git cd Teamwork
TIP: Todos trabajan sobre el MISMO repositorio. No es necesario hacer fork — solo clonar directamente.
02

CREA TU BRANCH

Antes de hacer cambios, crea una rama nueva con el formato:

git checkout -b feature/pagina-nombre-apellido
IMPORTANTE: Usa minúsculas, guiones en vez de espacios, sin acentos. Ejemplo: feature/pagina-juan-perez
TIP: Tu branch es tu espacio de trabajo personal. Puedes hacer todos los commits que quieras sin afectar a los demás.
03

CREA TU AVATAR

Abre avatar-builder.html en tu navegador o haz clic en "CREA TU AVATAR" arriba. Diseña tu skin pixel-art y copia el código de 64 caracteres que genera.

TIP: También puedes usar un preset: "steve", "alex", "creeper", etc.
04

EDITA TEAM.JS

Abre js/team.js y busca el comentario "AGREGA TU LINEA AQUI ABAJO". Copia el template y llena todos los campos con tus datos.

{ name: "Tu Nombre", page: "pages/nombre-apellido.html", tagline: "Una frase sobre ti", avatar: "TU_CODIGO_DE_64_CHARS", github: "https://github.com/tu-user", about: "Un parrafo sobre ti.", skills: ["Skill1", "Skill2", "Skill3"], randomFact: "Dato curioso", biome: "forest" },
IMPORTANTE: NO olvides la coma al final del objeto. Sin ella, el código falla para TODOS.
05

CREA TU PAGINA PERSONAL

Copia el archivo pages/_plantilla.html y renómbralo con tu nombre. Ejemplo: pages/juan-perez.html.

IMPORTANTE: El nombre del archivo DEBE coincidir EXACTAMENTE con el campo page que pusiste en team.js.
TIP: No necesitas editar nada dentro del HTML — todo se carga automáticamente desde team.js.
06

(OPCIONAL) ELIGE TU BIOMA

En tu objeto de team.js, el campo biome define el fondo de tu página personal. Las opciones son:

biome: "forest" // Bosque (default) biome: "desert" // Desierto biome: "jungle" // Selva tropical biome: "taiga" // Taiga / Invierno biome: "plains" // Praderas / Otoño biome: "nether" // El Nether biome: "end" // The End
07

VERIFICA EN TU NAVEGADOR

Abre dashboard.html en tu navegador y verifica que tu tarjeta aparece en la galería. Luego abre tu página personal y revisa que todos tus datos se muestran correctamente.

TIP: Si ves "datos no encontrados", revisa que el campo page en team.js coincida exactamente con el nombre de tu archivo HTML.
08

HAZ COMMIT

Guarda tus cambios con un commit descriptivo:

git add . git commit -m "feat: agregar pagina de Juan Perez"
FORMATO: Usa el formato feat: descripcion en tus mensajes de commit. Lee CONVENTIONS.md para más detalles.
09

PUSH TU BRANCH

Sube tu branch al repositorio en GitHub:

git push origin feature/pagina-nombre-apellido
TIP: Si es la primera vez que subes tu branch, Git te pedirá que la crees en el remoto. Solo confirma y listo.
10

CREA TU PULL REQUEST

Ve al repositorio en GitHub. Verás un banner con tu branch reciente y un botón "Compare & pull request". Haz clic y envía tu PR para que el instructor lo revise.

TIP: Agrega un título como "feat: agregar página de Juan Pérez" y una descripción breve de lo que hiciste. Espera la aprobación antes de mergear.
